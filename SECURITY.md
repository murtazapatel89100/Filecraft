# Security Policy

## Supported Versions

Security fixes are applied to the latest release on `main`.

## Reporting a Vulnerability

Please **do not** open public issues for suspected vulnerabilities.

Instead, report privately by email:

- **Contact:** murtazapatel89100@gmail.com
- **Subject:** `[organizer-cli security] <short summary>`

Include the following details when possible:

- affected implementation (`file-organiser-python`, `file-organiser-go`, or both)
- affected version/tag
- impact and attack scenario
- reproduction steps or proof of concept
- suggested remediation (if available)

## Response Timeline

- Initial acknowledgement target: within 72 hours
- Triage and severity assessment: as soon as possible
- Fix/release timeline: based on severity and reproducibility

## Disclosure

We follow coordinated disclosure. Once a fix is available, a release will be published and notes will include security context where appropriate.
